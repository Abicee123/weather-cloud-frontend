User Guide: Parameter Thresholds & Graph Logic

MQ135 Gas (PPM)

Measures volatile organic compounds (VOCs) including smoke, ammonia, benzene, and exhaust gases. Evaluated in Parts Per Million (PPM), meaning the number of toxic molecules per one million air molecules.

  • 0 - 450 PPM (Safe): Normal, clean fresh air baseline.
  • 450 - 600 PPM (Warning): Moderate pollution. Feels stuffy; prolonged exposure may irritate asthma.
  • Above 600 PPM (Hazard): Critical danger limit. Severe chemical or smoke presence requiring immediate action.

Particulate Dust (mg/m³)

Optical sensors utilize an infrared LED to measure physical solid particles floating in the air (dirt, ash, fine smoke). Evaluated in Milligrams per Cubic Meter (mg/m³).

  • 0.00 - 0.05 mg/m³ (Clear): Optimal, clean environment.
  • 0.06 - 0.10 mg/m³ (Hazy): Noticeable dust presence, potential respiratory irritant.
  • Above 0.10 mg/m³ (Severe): Dense particulate cloud. N95 filtration masks recommended.

The Unification Scaling Rule: How we plot mg/m³ and PPM together

Plotting multiple environmental factors on a single Y-Axis creates a massive mathematical visualization problem. A dangerous Gas reading is 600 (PPM), while a dangerous Dust reading is 0.10 (mg/m³). If we plot them normally, the dust line would be invisible—completely flattened against the bottom of the graph at 0.10.

To solve this, our JavaScript visualization engine applies a x1000 Multiplier to all incoming dust arrays strictly for rendering purposes. Therefore, a reading of 0.05 mg/m³ is plotted on the graph at 50. This mathematical normalization allows the human eye to instantly track the momentum and relationship between gas and dust simultaneously without requiring separate, confusing Y-axes.

Algorithm Architecture: Why do we use two models?

Our system architecture explicitly prevents single-point computational failures by utilizing two distinct, competing predictive algorithms simultaneously. Each excels in a different domain, ensuring maximum human safety regardless of the timeframe or network status.

Edge AI: Holt's Method (The Sprinter)

  • How it works: Runs locally in the web browser using pure Javascript. It calculates both the immediate Level and the current Velocity (Trend) of the gas.
  • Why it is best for SHORT-TERM: Instant execution. If a sudden chemical fire starts, Holt's Method instantly catches the massive velocity spike and draws a sharp warning line upward, alerting users seconds before the actual gas reaches dangerous levels.
  • Weakness: It is mathematically rigid. It cannot predict long-term weather patterns or how parameters interact with each other.

Cloud AI: Polynomial ML (The Marathoner)

  • How it works: Runs on a dedicated Python Flask server. Analyzes thousands of historical data points using a 3rd-Degree Scaled Polynomial Ridge Regressor.
  • Why it is best for LONG-TERM: True non-linear understanding. The ML model understands complex environmental physics (e.g., high humidity traps dust). It draws smooth, highly accurate curved projections into the future to predict hourly clearance rates.
  • Weakness: Requires cloud server access. Takes slightly longer to compute massive datasets than the lightweight Edge AI.
Conclusion: Edge AI (Holts) guarantees instant emergency response, while Cloud AI (ML) provides accurate long-term environmental forecasting.

How Our System Works

Step 1: Hardware Data Capture

The Transmitter ESP32 collects analog signals from the MQ135 and Sharp dust sensor. It packages these values and broadcasts them via LoRa radio frequencies. The Receiver ESP32 acts as a gateway, catching these LoRa packets over several kilometers.

Step 2: Database Ingestion (MongoDB)

The Receiver ESP32 connects to Wi-Fi and pushes the packets directly into a **MongoDB Atlas** cloud database, ensuring permanent data retention and completely eliminating the "single point of failure" common in local server setups.

Step 3: Backend API (Python)

A **Python Flask** server acts as a middleman. It securely reads the database and serves the sanitized JSON data to the website, meaning the frontend dashboard never holds the sensitive database credentials.

Step 4: Dual-Layer Machine Learning Forecasting

Instead of just showing past data, our system proactively predicts the future to protect public safety. We engineered two entirely distinct algorithms that run simultaneously:

Layer 1: Holt's Method (Edge AI / Short-Term)

Executed instantly within the user's web browser using native JavaScript. Simple averages fail to warn users quickly because they look backwards. Holt's Double Exponential Smoothing solves this by mathematically calculating both the Base Level and the Momentum (Velocity) of the gas, allowing the AI to plot an immediate warning spike before the gas even arrives.

The Equations:

  • 1. Level ($L_t$): Lt = αYt + (1-α)(Lt-1 + Tt-1)
  • 2. Trend ($T_t$): Tt = β(Lt - Lt-1) + (1-β)Tt-1
  • 3. Forecast ($F_{t+k}$): Ft+k = Lt + kTt

Where α (Alpha) = 0.5 ensures the AI prioritizes recent sensor spikes over historical data to prevent lag, and β (Beta) = 0.3 filters out minor sensor noise while preserving the true momentum curve.

Layer 2: Polynomial Ridge Regression (Cloud AI / Long-Term)

Executed remotely on our heavy-duty Python Flask server using `scikit-learn`. Unlike Holt's method, which is linear and rigid, Polynomial Regression is highly flexible and capable of understanding complex, non-linear environmental physics.

  • Massive Data Ingestion: The model fetches and trains on up to 2,500 historical data points per request to establish deep baseline curves.
  • Multi-Variable Awareness: It evaluates how variables interact (e.g., recognizing that a drop in Temperature combined with high Humidity will severely trap particulate Dust).
  • 3rd-Degree Extrapolation: By structuring the math as a 3rd-Degree Polynomial `(ax³ + bx² + cx + d)`, the AI can project smooth, highly accurate "S-Curves" representing the hourly atmospheric clearance rate for the next 6 hours.
