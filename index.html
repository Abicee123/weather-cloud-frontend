System Architecture & AI Forecasting

1. The Problem

Standard environmental monitors rely on short-range Wi-Fi (ESP8266/NodeMCU) or Bluetooth. This renders them ineffective for large-scale deployments across industrial zones, dense forests, or expansive college campuses where standard internet infrastructure is absent.

2. The LoRa Hardware Bridge

We engineered a robust hardware pipeline utilizing Long Range (LoRa) RF technology.

  • Transmitter Node: An ESP32 interfaces with MQ135 (Gas) and Sharp (Dust) sensors. It packages this raw analog data into JSON payloads and transmits it via a LoRa transceiver over kilometers of open space.
  • Gateway Receiver: A second ESP32 acts as the internet gateway. It catches the LoRa packets and utilizes its onboard Wi-Fi to push the payloads directly into our cloud database, bypassing local network constraints at the sensing site.

3. Cloud & Middleware

  • MongoDB Atlas: Acts as our highly scalable NoSQL data lake, capable of handling millions of real-time telemetry rows.
  • Python Flask API: A lightweight middleware server securely authenticates with MongoDB, parses the ObjectIDs, formats timestamps, and broadcasts the sanitized data locally via a REST API endpoint.

4. Predictive AI (Holt's Double Exponential Smoothing)

Instead of relying on simple linear regression—which fails to account for sudden momentum shifts in weather and gas dispersal—our frontend executes Holt's Linear Trend Method natively in JavaScript. This allows us to forecast impending hazardous conditions before they reach critical thresholds.

The Mathematics

1. Level: Lt = αYt + (1-α)(Lt-1 + Tt-1)

2. Trend: Tt = β(Lt - Lt-1) + (1-β)Tt-1

3. Forecast: Ft+k = Lt + kTt

α (Alpha): Controls how much weight is given to the absolute newest sensor reading.

β (Beta): Controls how quickly the algorithm adapts to changes in the *trend* (e.g., a sudden spike in gas).

This runs seamlessly on the client side, producing the dotted trajectory lines on the live feed.
